## TL;DR

- 2026年4月22日、Cloudflare公式ブログで **「Agents + Stripe Projects」** 統合が発表
- そして **本日4月30日、ついに本番提供が開始**
- AIエージェント（Cursor / Claude / Windsurf など）が、人間の手を介さずに **Cloudflareアカウント新規作成 → 有料プラン契約 → ドメイン購入 → APIトークン取得 → 本番デプロイ** まで一気通貫で実行できるようになった
- 人間がやるのは **Stripe CLIで数コマンド叩く + OAuthで一度だけ承認** だけ
- さらに、Stripe Atlasで法人化した新規スタートアップには **Cloudflareクレジット$100,000** が付与されるおまけ付き

僕もStripeアカウントを持っていたので、本番提供開始当日に試してみました。素直に「未来きた」という感想です。

## 何が変わったのか

これまでAIエージェントに「Webアプリを作ってデプロイしておいて」と頼んでも、どうしても人間の作業が残っていました。Cloudflareアカウントの作成、クレジットカード情報の入力、APIトークンのコピペ、ドメインの購入──そのどれもが「エージェントの自動化フロー」を分断する原因でした。

今回の統合で、その分断がほぼなくなります。

| 項目 | 従来 | Stripe Projects導入後 |
| --- | --- | --- |
| アカウント作成 | 人間が手動 | エージェントが自動 |
| 支払い情報入力 | 人間が手動 | Stripeがトークン化して安全に処理 |
| APIトークン発行 | 手動でコピペ | 自動取得・自動保存 |
| ドメイン購入 | 人間が手動 | Cloudflare Registrarで自動購入 |
| 所要時間 | 10〜30分 | 数分（OAuth承認1回のみ） |

特筆すべきは、**カード番号やAPIキーがエージェントに直接渡らない** という設計です。Stripeが支払いを仲介し、Cloudflareへのアクセスは一時的なOAuthトークンで行われるため、エージェントに過剰な権限を渡さずに自動化できます。

## 必要なもの

- Stripeアカウント（支払い方法登録済み）
- Stripe CLI
- AIコーディングエージェント（Cursor / Claude Code / Windsurf など、Stripe Projects のスキルに対応したもの）

## セットアップ手順

### 1. Projectsプラグインをインストール

```bash
stripe plugin install projects
```

`stripe projects --help` が通れば成功です。

### 2. Stripeにログイン

```bash
stripe login
```

ブラウザが開くので認証してください。

### 3. プロジェクトを初期化

```bash
stripe projects init my-app
```

これがいちばん大事なステップです。実行すると、カレントディレクトリに `.projects/` フォルダと **AIエージェント用のスキルファイル** が生成されます。これでエージェント側が `stripe projects` の使い方を「理解」した状態になります。

### 4. 利用可能なサービスを確認（任意）

```bash
stripe projects catalog
```

ホスティング、データベース、認証、AI、解析など、対応プロバイダーが一覧表示されます。

### 5. Cloudflareサービスを追加

```bash
# ホスティング（Workers / Pages）
stripe projects add cloudflare/hosting

# 独自ドメインも欲しい場合
stripe projects add cloudflare/registrar:domain
```

Stripeのログインメールに紐づくCloudflareアカウントが **存在しなければ自動作成**、存在すればOAuth画面が出て、ワンクリックでエージェントにアクセスを許可できます。

支払い方法をまだ登録していない場合は次を実行：

```bash
stripe projects billing add
```

## エージェントへの指示例

ここまで終われば、あとはエージェントに自然言語で指示するだけです。Cursor / Claude Code などにそのまま貼り付けて使えます。

> 「Stripe Projectsを使って、新しいCloudflareプロジェクトをセットアップしてください。シンプルなNext.jsアプリを用意して、独自ドメインを購入し、本番にデプロイするところまで自動でお願いします。」

エージェントは内部で順に：

1. `stripe projects catalog` で対応サービスを確認
2. `stripe projects add cloudflare/hosting` でホスティング契約
3. `stripe projects add cloudflare/registrar:domain` でドメイン購入
4. Wrangler を使ってビルド & デプロイ

を実行します。各ステップで課金や購入が発生するときは、Stripe側からユーザーに承認を求めるフローが入るので、暴走の心配はありません。

## 注意点・運用Tips

- **支出上限**: デフォルトでは **1プロバイダーあたり $100/月** に制限されます。引き上げたい場合は `stripe projects billing` から設定。
- **クレデンシャルの扱い**: APIキーやトークンは `.projects/` 配下にローカル保存されます。`vault/` や `.env` を含むディレクトリは **必ず `.gitignore` に追加**。`.projects/state.json` は環境定義としてコミットしてOK。
- **CI/CD用途**: 非対話モードで動かしたい場合は `--no-interactive --auto-confirm` を付与。
- **既存のCloudflareアカウントを使いたい**: Stripeのログインメールと同じメールでCloudflareにサインインしておけば、OAuthで自動連携されます。
- **ロールバック**: 不要になったサービスは `stripe projects remove <provider>/<service>` で解除可能。サブスクリプションも合わせて停止されます。

## 試してみた感想

「Next.jsの簡単なアプリを作ってデプロイして」と一言投げてから、独自ドメインで本番URLが返ってくるまで **おおよそ数分**。途中、ドメイン購入とサブスクリプション開始の確認ダイアログが計2回出ただけで、それ以外は完全にエージェント任せでした。

特にインディーハッカーや個人開発者にとって、「アイデア → 公開URL」の摩擦がここまで減るのはインパクトが大きいです。**プロトタイプを試す心理的コストが、ほぼゼロになる感覚** に近いです。
