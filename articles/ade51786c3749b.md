# AIキャラクターに口パクさせた話

ローカルLLMに話しかけると、Live2Dキャラクターが**音声に同期して口を動かし、感情に合わせて表情も変わる**——そんな機能を自作AIチャットアプリ「Voice Bridge」に実装しました。

この記事では、その設計・実装・ハマりポイントをすべて共有します。

## できあがったもの

- マイクで話しかける
- ローカルLLM（Ollama）が返答を生成
- VOICEVOXやCoeiroInkで音声合成
- **Live2DキャラクターがTTSに合わせてリアルタイムで口パク**
- テキスト内容から感情を推定して**表情も自動で変化**

リポジトリ：https://github.com/zephel01/voice-bridge

---

## システム全体アーキテクチャ

まず全体像から把握しましょう。

```
voice-bridge (Python)
  ├─ ASR（Whisper / Moonshine / Qwen3）
  ├─ LLM（Ollama ローカル）
  ├─ TTS（VOICEVOX / CoeiroInk / Edge TTS）
  │      │
  │      ▼ mp3生成
  └─ live2d_bridge.py ── WebSocket(8765) ──▶  live2d-ui (Electron)
                                               ├─ pixi-live2d-display でモデル表示
                                               ├─ Web Audio API で mp3 再生
                                               ├─ AnalyserNode → リップシンク
                                               ├─ AutoBlink / AutoBreath
                                               └─ emotionMap.ts で感情表情補間
```

**設計のポイント**は「音声再生をElectron側に任せること」です。Pythonでmp3ファイルを生成したら、そのバイナリをWebSocket経由でElectronに送信し、フロントのWeb Audio APIで再生します。こうすることで、`AnalyserNode`でリアルタイムに音量データを取得でき、音声と口パクのズレがなくなります。

## 実装ステップ

### Step 1：Python側 — `live2d_bridge.py`

Python側のWebSocketサーバーを作ります。役割は2つ：
1. Electronクライアントが接続したら「TTS音声 + テキスト + 感情」を送信する
2. Electronからの再生完了通知を受け取って次の処理を進める

```python
# live2d_bridge.py（概略）
import asyncio
import websockets
import json, base64, uuid
from pathlib import Path

class Live2DBridge:
    def __init__(self, host="127.0.0.1", port=8765):
        self._host = host
        self._port = port
        self._client = None          # 接続中のElectronクライアント
        self._pending: dict = {}     # 再生完了を待つIDのセット
        self._loop = None
        self._server_task = None

    def start(self):
        """バックグラウンドスレッドでWebSocketサーバーを起動"""
        import threading
        t = threading.Thread(target=self._run, daemon=True)
        t.start()

    def _run(self):
        self._loop = asyncio.new_event_loop()
        asyncio.set_event_loop(self._loop)
        self._loop.run_until_complete(self._serve())

    async def _serve(self):
        async with websockets.serve(self._handler, self._host, self._port):
            await asyncio.Future()  # 永続起動

    async def _handler(self, ws):
        self._client = ws
        try:
            async for raw in ws:
                msg = json.loads(raw)
                if msg.get("type") == "playback_end":
                    pid = msg.get("id")
                    if pid in self._pending:
                        self._pending[pid].set()
        finally:
            self._client = None

    def has_client(self) -> bool:
        return self._client is not None

    def send_tts(self, audio_path: str, text: str = "",
                 emotion: str = "neutral", intensity: float = 0.5) -> str | None:
        """mp3ファイルをBase64エンコードしてElectronに送信"""
        if not self.has_client():
            return None
        pid = str(uuid.uuid4())
        audio_b64 = base64.b64encode(Path(audio_path).read_bytes()).decode()
        payload = json.dumps({
            "type": "tts",
            "id": pid,
            "audio": audio_b64,
            "text": text,
            "emotion": emotion,
            "intensity": intensity,
        })
        ev = asyncio.Event()
        self._pending[pid] = ev
        # スレッドセーフな送信
        asyncio.run_coroutine_threadsafe(
            self._client.send(payload), self._loop
        )
        return pid

    def wait_playback_end(self, pid: str, timeout: float = 30.0):
        """再生完了を同期的に待つ"""
        ev = self._pending.get(pid)
        if ev:
            ev.wait(timeout=timeout)
            self._pending.pop(pid, None)
```

### Step 2：感情推定 — `infer_emotion()`

テキストから感情タグを推定する軽量関数を用意します。LLMを使わず、キーワードマッチで十分実用になります。

```python
def infer_emotion(text: str) -> tuple[str, float]:
    """
    テキストから感情タグと強度を返す。
    Returns: (emotion, intensity)
    """
    text_lower = text.lower()
    rules = [
        (["嬉しい", "よかった", "やった", "最高", "！", "笑"], "happy",   0.8),
        (["悲しい", "つらい", "残念", "ごめん"],                "sad",     0.7),
        (["怒", "むかつく", "ふざけ"],                          "angry",   0.7),
        (["驚", "えっ", "マジ", "！？"],                        "surprise",0.6),
        (["照れ", "えへ", "恥ずか"],                            "shy",     0.5),
    ]
    for keywords, emotion, intensity in rules:
        if any(k in text for k in keywords):
            return emotion, intensity
    return "neutral", 0.3
```

### Step 3：`main.py` への統合

既存のTTS → pygame再生フローを壊さないよう、`_play_or_forward()`というラッパーを追加します。`--live2d`フラグがOFFのときは従来どおりpygameで再生されます。

```python
# main.py（追加部分）
from live2d_bridge import Live2DBridge, infer_emotion

class VoiceBridge:
    def __init__(self, ..., live2d_enabled=False,
                 live2d_host="127.0.0.1", live2d_port=8765):
        # ...既存コード...
        self.live2d: Live2DBridge | None = None
        if live2d_enabled:
            self.live2d = Live2DBridge(host=live2d_host, port=live2d_port)
            self.live2d.start()

    def _play_or_forward(self, audio_path: str, text: str = ""):
        """Live2D有効時はElectronへ転送、無効時はpygameで再生"""
        if not audio_path:
            return
        if self.live2d and self.live2d.has_client():
            emotion, intensity = infer_emotion(text)
            pid = self.live2d.send_tts(
                audio_path, text=text,
                emotion=emotion, intensity=intensity
            )
            if pid:
                self.live2d.wait_playback_end(pid, timeout=30.0)
            try:
                import os; os.remove(audio_path)
            except OSError:
                pass
            return
        # 通常モード
        self.player.enqueue(audio_path)
```

あとは既存の `self.player.enqueue(audio_path)` 呼び出しを
`self._play_or_forward(audio_path, text=text)` に置き換えるだけです。

### Step 4：Electron側 — `Live2DAvatar.ts`

Electronフロントの核心部分です。pixi-live2d-displayでモデルを表示し、Web Audio APIのAnalyserNodeで音量をリアルタイムに取り出してパラメータに反映します。

```typescript
// Live2DAvatar.ts（概略）
import * as PIXI from "pixi.js";
import { Live2DModel } from "pixi-live2d-display/cubism4";

export class Live2DAvatar {
  private app: PIXI.Application;
  private model: Live2DModel | null = null;
  private analyser: AnalyserNode | null = null;
  private mouthParamId: string;
  private lipSyncGain: number;

  constructor(opts: {
    canvas: HTMLCanvasElement;
    modelUrl: string;
    mouthParamId?: string;
    lipSyncGain?: number;
    scale?: number;
  }) {
    this.mouthParamId = opts.mouthParamId ?? "ParamMouthOpenY";
    this.lipSyncGain  = opts.lipSyncGain  ?? 1.6;

    this.app = new PIXI.Application({ view: opts.canvas, transparent: true });
    this._loadModel(opts.modelUrl, opts.scale ?? 0.28);
    this._startTick();
  }

  private async _loadModel(url: string, scale: number) {
    this.model = await Live2DModel.from(url);
    this.model.scale.set(scale);
    this.app.stage.addChild(this.model as unknown as PIXI.DisplayObject);
    // 画面中央に配置
    this._applyLayout();
  }

  /** TTS音声（Base64）を受け取って再生しつつリップシンク */
  async playAudio(base64mp3: string): Promise<void> {
    const ctx = new AudioContext();
    const buf = Uint8Array.from(atob(base64mp3), c => c.charCodeAt(0));
    const decoded = await ctx.decodeAudioData(buf.buffer);

    this.analyser = ctx.createAnalyser();
    this.analyser.fftSize = 256;

    const source = ctx.createBufferSource();
    source.buffer = decoded;
    source.connect(this.analyser);
    this.analyser.connect(ctx.destination);
    source.start();

    return new Promise(resolve => {
      source.onended = () => {
        this.analyser = null;
        resolve();
      };
    });
  }

  /** 毎フレーム：音量 → 口パクパラメータに反映 */
  private _startTick() {
    this.app.ticker.add(() => {
      if (!this.model || !this.analyser) return;
      const data = new Uint8Array(this.analyser.frequencyBinCount);
      this.analyser.getByteFrequencyData(data);
      const avg = data.reduce((s, v) => s + v, 0) / data.length;
      const mouthOpen = Math.min(1, (avg / 128) * this.lipSyncGain);
      this.model.internalModel.coreModel.setParameterValueById(
        this.mouthParamId, mouthOpen
      );
    });
  }
}
```

### Step 5：感情プリセット — `emotionMap.ts`

感情タグに対応するLive2Dパラメータの差分値を定義します。モデルによってパラメータIDが異なるので、ここを調整することでどんなモデルにも対応できます。

```typescript
// emotionMap.ts
export type EmotionPreset = {

  duration: number;                  // 補間時間 (ms)
};

export const EMOTION_PRESETS: Record<string, EmotionPreset> = {
  neutral: {
    params: { ParamMouthForm: 0, ParamEyeLSmile: 0, ParamEyeRSmile: 0 },
    duration: 500,
  },
  happy: {
    params: { ParamMouthForm: 1.0, ParamEyeLSmile: 1.0, ParamEyeRSmile: 1.0 },
    duration: 300,
  },
  sad: {
    params: { ParamMouthForm: -1.0, ParamBrowLAngle: -0.5, ParamBrowRAngle: -0.5 },
    duration: 600,
  },
  angry: {
    params: { ParamBrowLAngle: -1.0, ParamBrowRAngle: -1.0, ParamMouthForm: -0.5 },
    duration: 200,
  },
  surprise: {
    params: { ParamEyeLOpen: 1.0, ParamEyeROpen: 1.0, ParamMouthOpenY: 0.5 },
```

### Step 6：WebSocket受信 — `App.tsx`

ElectronのフロントでWebSocketを受信し、音声再生 → 感情適用 → 再生完了通知の流れを作ります。

```typescript
// App.tsx（WebSocket処理部分）
useEffect(() => {
  const ws = new WebSocket("ws://127.0.0.1:8765");

  ws.onmessage = async (event) => {
    const msg = JSON.parse(event.data);
    if (msg.type !== "tts" || !avatarRef.current) return;

    // 1. 感情を適用
    avatarRef.current.setEmotion(msg.emotion, msg.intensity);

    // 2. 音声再生（リップシンク込み）
    await avatarRef.current.playAudio(msg.audio);

    // 3. 再生完了をPythonに通知
    ws.send(JSON.stringify({ type: "playback_end", id: msg.id }));
  };

  ws.onopen  = () => setStatus("connected");
  ws.onclose = () => setStatus("disconnected");

  return () => ws.close();
}, []);
```

## ハマりポイントと解決策

### 1. 口パクが全く動かない

**原因**：`mouthParamId` がモデルのパラメータIDと不一致。

Live2Dモデルのパラメータ名はモデル作者によって異なります。Cubism標準では `ParamMouthOpenY` ですが、一部モデルでは `PARAM_MOUTH_OPEN_Y` のようにフォーマットが違います。

**解決策**：ElectronのDevToolsコンソールでパラメータ一覧を確認します。

```javascript
// DevToolsコンソールで実行
const m = document.querySelector("canvas")?.__pixi?.stage?.children?.[0]
            ?.internalModel?.coreModel;
for (let i = 0; i < m.getParameterCount(); i++)
  console.log(m.getParameterId(i));
```

確認したIDを `Live2DAvatar` の `mouthParamId` に指定すればOKです。

### 2. 音声が2重に鳴る

**原因**：`--live2d` 指定時も `self.player.enqueue()` が呼ばれている場所がある。

**解決策**：`main.py` 内の `self.player.enqueue(audio_path)` をすべてgrepして `_play_or_forward()` に置換します。

```bash
grep -n "player.enqueue" main.py
```

### 3. WebSocketに「接続済み」と出るのにElectronに何も届かない

**原因**：`has_client()` が `True` を返す前にPython側が送信しようとしている。

`Live2DBridge._handler()` でクライアントが接続した瞬間に `self._client = ws` を設定していますが、Electronの初期化が完了していない段階でTTSが走ると届きません。

**解決策**：Electronを先に起動し、右上のステータスバッジが **Connected** になってからPythonを起動する。

### 4. 感情表情が変わらない

**原因**：`emotionMap.ts` のパラメータIDがモデルに存在しない。

**解決策**：DevToolsでパラメータ一覧を確認して `emotionMap.ts` のキーを書き換えます。特に頬の赤み（`ParamTere` vs `ParamCheek`）はモデルによって名前が違います。

### 5. Cubism Core のロードエラー

**原因**：`live2dcubismcore.min.js` が `public/` 直下に配置されていない、または `index.html` の `<script>` タグが壊れている。

**解決策**：
1. Live2D公式SDKをダウンロード: https://www.live2d.com/sdk/download/web/
2. 解凍して `Core/live2dcubismcore.min.js` を `live2d-ui/public/` にコピー
3. DevToolsコンソールに `Live2D Cubism Core version: ...` が表示されることを確認

## まとめ

今回の実装でポイントになったのは以下の3点です。

**① 音声再生をフロントに委ねる設計**
PythonでTTSファイルを生成したら、再生はElectronに任せます。AnalyserNodeでリアルタイム音量を取れるため、音声と口パクがぴったり合います。

**② フォールバック設計**
`--live2d` フラグなしのときはpygameで通常再生、ありのときはElectronに転送——という分岐を1つのラッパー関数にまとめることで、既存の動作を壊さずに機能追加できました。

**③ モデル非依存の設計**
パラメータIDを外部設定（`mouthParamId`, `emotionMap.ts`）で切り替えられるようにしたおかげで、どんなLive2Dモデルにも対応できます。
