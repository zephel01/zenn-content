## この記事のゴール

note / Qiita / Zenn を並行して書いている人向けに、3媒体の執筆ワークフローを1つにまとめたリポジトリ **[oasis-article](https://github.com/zephel01/oasis-article)** を作ったので、その使い方と中身を紹介します。

- 🌴 **1本の下書きを媒体ごとに最適化**してリライトできる
- 🦙 **ローカル LLM (Ollama) で完結** — 外部 API にデータを出さない
- 🧽 **AI 臭 40+ パターンを自動検出**して自分の文体に戻す

:::message
この記事は自分がリライトツール `article_rewriter.py` を作りながら気付いたことの記録でもあります。コード中心ではなく、**「なぜこの構成にしたか」と「ローカル LLM でどこまで実用になるか」** に重きを置いています。
:::

## 何に困っていたか

最近の個人執筆あるある。

- ChatGPT や Claude に下書きをさせると**「いかがでしたか？」問題**が発生する
- 同じネタを **note に柔らかく、Qiita に硬く、Zenn に記法たっぷり**で書き直したい
- でも毎回手で書き直すのは怠い
- かといって **Claude に「Qiita 用に直して」と頼むと別の AI 臭が乗る**

要は「ローカルで・自分の文体で・媒体別に」リライトしたい。その道具箱がなかったので作りました。

## 構成

```
oasis-article/
├── 🛠️  tools/    ← 3媒体共通のローカル LLM ツール群
├── 📝  note/     ← note.com 記事ワークスペース
├── 📚  qiita/    ← Qiita CLI ワークスペース
└── ⚡  zenn/     ← Zenn CLI ワークスペース（この記事もここ）
```

各媒体は独立した CLI（`zenn-cli`, `@qiita/qiita-cli`）のワークスペースとして動くので、従来の書き方をそのまま維持できます。追加要素は `tools/` 配下の **共通リライター** だけ。

https://github.com/zephel01/oasis-article

## キモ 1：ローカル LLM でリライトする

`tools/article_rewriter.py` は **Ollama 経由でローカル LLM を呼ぶ** だけの CLI です。

```bash
# note の記事URLから取得してリライト
python tools/article_rewriter.py https://note.com/zephel01/n/nXXXX --platform note

# ローカルの Markdown を食わせる
python tools/article_rewriter.py draft.md --platform qiita
```

:::message
**外部 API には一切送らない**。記事の下書きは書き途中で見せたくないことが多いので、これは譲れない要件でした。Ollama を使えば GPT-4o 並の API を使わずとも、**実用十分な品質**にリライトできます。
:::

### 推奨モデル（32 GB 以内）

現実的に個人で回せる範囲に絞るとこのあたり。

| メモリ | モデル | サイズ目安 |
| --- | --- | --- |
| 8〜16 GB | `qwen2.5:7b-instruct` / `gemma2:9b-instruct` | 5〜6 GB |
| 16〜24 GB | **`qwen2.5:14b-instruct`** ⭐ | 9〜11 GB |
| 24〜32 GB | `qwen2.5:32b-instruct-q4_K_M` | ~20 GB |

迷ったら **Qwen2.5 14B** から。日本語の自然さは Gemma 2 27B（Q4〜Q5）が頭ひとつ抜けますが、命令追従の素直さでは Qwen2.5 系が強いです。

## キモ 2：AI 臭 40+ パターンを検出する

LLM に書かせた文章には癖があります。`article_rewriter.py` は**パターンマッチで検出**し、LLM でリライトするか、`--deai` モードで「ここが怪しい」と指摘だけさせることもできます。

```bash
# LLM を呼ばず、AI 臭の検出だけ
python tools/article_rewriter.py draft.md --deai
```

検出対象の一部を抜粋するとこんな感じです。

:::details 検出する AI 臭パターン（抜粋）

| 元表現 | 置換候補 |
| --- | --- |
| 〜と言えるでしょう | 断定 or 感想に |
| 本記事では〜ご紹介します | 削除、いきなり本題 |
| いかがでしたか？ | 全削除 |
| 〜することが可能です | 〜できる |
| 〜を活用することで | 〜を使えば |
| 〜が求められます | 〜が必要 |
| 〜は非常に重要です | 〜が大事 |
| 素晴らしい質問ですね | 全削除（ChatGPT 特有） |
| お役に立てれば幸いです | 全削除 |

ChatGPT / Claude / Gemini / Grok それぞれの**モデル特有の癖**も別テーブルで持っています。
:::

## キモ 3：`--platform` で媒体ごとに最適化する

同じ下書きでも、媒体ごとに記法と空気感が違います。

- **note**: `:::message` や `<details>` は効かないので、装飾は見出しと箇条書きのみ
- **Qiita**: `> **Note**:` ブロックや `<details>` で情報の階層を作る
- **Zenn**: `:::message` `:::details` `@[card]` `@[tweet]` などを積極活用

`--platform` フラグを渡すと、媒体固有のスタイルテンプレと**システムプロンプトに注入する記法ルール**が自動で切り替わります。

```bash
# 1本の下書きを3媒体に振り分ける
python tools/article_rewriter.py draft.md --platform note  --output note/articles/draft
python tools/article_rewriter.py draft.md --platform qiita --output qiita/public/draft
python tools/article_rewriter.py draft.md --platform zenn  --output zenn/articles/draft
```

内部的には `tools/templates/<platform>.md` を読み込んでシステムプロンプトに埋め込むだけの素朴な仕組みですが、Zenn の `:::message` を Qiita 向け出力に混ぜてしまう、のような事故を防げます。

## 技術的に気をつけた点

### コードブロックを LLM に触らせない

LLM にリライトさせると、**コードブロックの中の変数名まで勝手に書き換える**ことがあります。これを防ぐため、リライト前にコードブロックを `【CODE_BLOCK_N】` のようなプレースホルダに置換し、リライト後に復元しています。

```python:article_rewriter.py(抜粋)
class CodeBlockProtector:
    def protect(self, text: str) -> str:
        # ``` ... ``` を検出してプレースホルダ化
        ...
    def restore(self, text: str) -> str:
        # リライト後に元のブロックを差し戻す
        ...
```

さらに**差分検証**も入れていて、復元後にコードブロックが変化していたら原文に差し戻す二重ガードにしています。

### セクション分割と反復検出

ローカル LLM は長文だと途中で同じ文を繰り返す（ストリーミング反復）ことがあります。対策として、

- セクション（`##` 見出し単位）で分割してリライト
- チャンクサイズはモデルサイズから自動推定
- ストリーミング中に繰り返しを検出したら早期中断

を入れています。`--chunk-size` で上書きもできます。

## 使い始め方

最短 3 ステップです。

```bash
# 1. clone
git clone https://github.com/zephel01/oasis-article
cd oasis-article

# 2. Ollama とモデル
brew install ollama
ollama pull qwen2.5:14b-instruct
ollama serve

# 3. Python 依存
cd tools && pip install -r requirements.txt && cd ..

# 使ってみる
python tools/article_rewriter.py draft.md --platform zenn --deai
```

`--deai` だけなら Ollama すら不要で、純粋なパターンマッチで AI 臭だけ炙り出せます。**「とりあえず自分の書いた文章が AI くさくないか確認したい」** 用途にも使えます。
