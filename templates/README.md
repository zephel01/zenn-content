# templates/

Zenn 記事を書くときのベース原稿テンプレートを置いています。
このフォルダのファイルは **Zenn に直接公開するもの** ではなく、
手元の下書きの出発点として使うひな形です。

## 使い方

1. 書きたい内容に合ったテンプレをコピー
   ```bash
   # 例: tech記事を書きたい
   cp templates/tech-article.md ~/Desktop/my-draft.md
   ```
2. 好きなエディタで中身を書き換える（`<!-- -->` のコメントは消してOK）
3. リポジトリのルートで補助CLIを起動:
   ```bash
   npm run article:from-draft
   ```
4. 原稿ファイルに `~/Desktop/my-draft.md` を指定 → タイトル等を対話入力
5. `articles/<slug>.md` として取り込まれる

> front matter（`title:` や `topics:` など）はテンプレに **含めません**。
> 補助CLIが対話入力した値から自動で付与します。

## テンプレート一覧

| ファイル | 用途 | type | こんな時に使う |
|---------|------|------|----------------|
| [`tech-article.md`](./tech-article.md) | 汎用の技術記事 | tech | 問題 → 解決策 → コード → まとめ の王道構成 |
| [`tutorial.md`](./tutorial.md) | チュートリアル | tech | ステップバイステップで何かを作って完成させる記事 |
| [`idea-essay.md`](./idea-essay.md) | アイデア/ポエム | idea | 設計思想・体験談・キャリア話などの読み物 |
| [`multipart-series.md`](./multipart-series.md) | 連載の第1回 | tech | 複数記事に分けて 1 つのテーマを扱う時の導入編 |

## 迷ったら

- **手を動かして作る系** → `tutorial.md`
- **問題を1つ解決する系** → `tech-article.md`
- **考えを整理する・意見を書く系** → `idea-essay.md`
- **大きすぎて1記事に収まらない** → `multipart-series.md`

## テンプレに書かれている要素

どのテンプレにも以下の共通要素を入れてあります:

- リード文の指針（コメントで指示）
- Zenn独自の記法チート（末尾にまとめ、不要なら削除）
  - `:::message` / `:::message alert` メッセージボックス
  - `:::details` 折りたたみ
  - コードブロックのファイル名表示・diff表示
  - リンクカード（URLを1行で書くだけ）
  - 画像 + キャプション記法
  - Tweet / YouTube 等の埋め込み
  - 脚注 / 数式 / 目次

## 独自のテンプレを増やしたい

このフォルダに新しい `.md` を置くだけでOKです。補助CLI側は
特別な処理はしておらず、あくまで「人間が参照するひな形集」として扱われています。

既存テンプレを写経しつつ、自分のよく書くパターンに寄せて育てていくと便利です。
